# DeepSeek 余额查看工具 — 需求文档

## 1. 项目背景

用户希望在个人安卓手机（红米 K70，Xiaomi HyperOS）上拥有一款轻量级小工具，能够随时查看 DeepSeek API 账户余额，无需打开浏览器或第三方平台。

## 2. 用户故事

- 作为 DeepSeek API 用户，我希望打开 App 就能看到当前余额，以便快速了解账户可用额度。
- 作为多账户用户，我希望保存并切换多个 API key，以便查看不同账户的余额。
- 作为桌面小部件用户，我希望在手机桌面直接看到余额，减少操作步骤。
- 作为通知栏用户，我希望 optionally 在通知栏实时显示余额，方便随时查看。
- 作为安全敏感用户，我希望 API key 在本地加密存储，避免泄露。

## 3. 功能需求

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1 | App 启动后展示当前选中 key 的 DeepSeek 余额 | 高 |
| F2 | 支持添加、删除、切换多个 API key | 高 |
| F3 | 支持桌面小部件展示余额，并提供手动刷新按钮 | 高 |
| F4 | App 内可开启/关闭通知栏实时余额显示 | 中 |
| F5 | 支持设置自动刷新间隔（15/30/60 分钟） | 中 |
| F6 | 手动刷新余额 | 高 |
| F7 | 错误提示：网络异常、key 无效等 | 高 |

## 4. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF1 | 安全性 | API key 必须使用 Android Keystore + EncryptedSharedPreferences 加密存储 |
| NF2 | 隐私 | 通知在锁屏界面隐藏余额详情 |
| NF3 | 省电 | 后台刷新使用 WorkManager，避免常驻前台服务 |
| NF4 | 兼容性 | 最低支持 Android 8.0（API 26），适配 HyperOS |
| NF5 | 体积 | APK 尽量小巧，避免跨平台引擎 |

## 5. 约束

- 只面向 Android 平台。
- 用户不懂 Android 开发，代码与配置应清晰、可直接导入 Android Studio 运行。
- 不引入未要求的附加功能（如充值、历史记录、iOS 版本）。

## 6. 验收标准

1. 在红米 K70 / 模拟器上能正常安装并打开。
2. 输入有效 DeepSeek API key 后，主界面显示与官方接口一致的余额数据。
3. 添加多个 key 后，切换 key 能看到对应余额变化。
4. 桌面小部件能正确显示余额，点击刷新后更新。
5. 开启通知栏显示后，能在通知栏看到余额；关闭后通知消失。
6. 断网场景下给出友好的错误提示；恢复网络后可正常刷新。

## 7. 相关接口

- DeepSeek 官方余额接口：`GET https://api.deepseek.com/user/balance`
- 文档：[DeepSeek API Docs - Get User Balance](https://api-docs.deepseek.com/api/get-user-balance/)
